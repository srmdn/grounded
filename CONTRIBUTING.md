# Contributing

Grounded is a narrow project on purpose.

Contributions are welcome when they make the system sharper, more usable, or more reusable for solo builders. Contributions that make it broader, noisier, or more generic are less likely to fit.

## Good Contributions

- clarify existing docs
- add practical examples
- improve templates
- improve checklists
- tighten wording and structure
- add small artifacts that solve recurring solo-builder problems

## Contributions To Avoid

- generic productivity advice
- growth or marketing playbooks
- large app features before the docs-first system is mature
- broad startup content that weakens the project's focus

## Working Style

- keep changes minimal
- prefer structure over volume
- prefer concrete examples over abstract claims
- keep the repo useful to both humans and AI agents

## Before Opening a PR

- explain the problem the change solves
- explain why the current repo is insufficient
- explain why the proposed change fits Grounded's scope

If the contribution is mostly personal planning or rough notes, keep it out of the public repo.
