# Self-Use Case

Grounded should prove itself on Grounded.

This document applies the system to the project itself so the repo is not only theory.

## Project

Name: grounded

One-sentence description:
An open-source system for solo builders who need clearer judgment and a more disciplined path from idea to live app.

## Audience

Solo builders with limited budget, limited time, and too many tools.

## Problem

AI made it easier to produce software but easier to drift as well.

Many solo builders now have:

- more ideas than focus
- more tools than judgment
- more apparent momentum than evidence

Grounded exists to help them choose the next sane step.

## Smallest Useful Version

A public docs-first repo with enough practical structure that a solo builder can use it to clarify a project, define a milestone, ship something small, and review what to do next.

## Core User Flow

1. builder arrives with a vague idea
2. builder uses Grounded to clarify the problem and smallest useful version
3. builder defines milestone 1 with constraints
4. builder ships something small
5. builder reviews what changed and chooses the next step

## Must-Haves

- clear public positioning
- doctrine
- workflow
- canonical guide
- project brief template
- milestone review template
- deploy checklist
- at least one example walkthrough

## Not Now

- accounts
- SaaS dashboard
- AI chat assistant
- analytics layer
- heavy application code

## Milestone 1

Concrete outcome:
Create a genuinely usable public repo for the Grounded system.

Definition of done:

- a new visitor can understand what Grounded is
- a solo builder can follow the flow from idea to milestone
- the repo contains practical templates and checklists
- the system is demonstrated on at least one real example

## What Was Actually Built

Milestone 1 currently includes:

- README positioning
- philosophy
- workflow
- start-here doc
- canonical guide
- audience-fit doc
- example walkthrough
- project brief template
- milestone review template
- deploy checklist

## What This Proves

Grounded can already function as a reusable system without requiring a web app.

That is important because it keeps the project honest:

- usefulness first
- interface second

## What Still Needs Proof

- whether people besides the creator can use it without guidance
- whether it reduces drift in a second real project
- which parts deserve automation later
