# Workflow

Grounded uses a simple loop:

`idea -> scope -> milestone 1 -> deploy -> validate -> learn -> reprioritize -> next milestone`

## 1. Idea

Start with three lines:

- who is this for
- what painful problem does it solve
- what is the smallest useful version

If those lines are vague, the project is not ready.

## 2. Scope

Turn the idea into a spec-lite:

- core user flow
- must-have features
- not-now features
- stack choice
- first milestone

This is planning to a solid enough state, not writing a giant document.

## 3. Milestone 1

Define the smallest meaningful outcome.

Examples:

- user can submit one form successfully
- page loads and data persists locally
- one recurring task is automated end to end

## 4. Deploy

Do not treat deployment as the start of thinking.
Use deployment as the point where verified work becomes available.

## 5. Validate

You do not always need real users first, but you do need signal:

- your own repeated use
- trusted tester feedback
- evidence that the main task is actually easier

## 6. Learn

After each milestone, ask:

- what worked
- what confused people
- what assumption failed
- what is the next bottleneck to usefulness

## 7. Reprioritize

Do not ask "what else can we add?"

Ask "what is the next sane step?"
