# Example Walkthrough

This is a worked Grounded example from vague idea to first shipped version.

The point is not the specific product.
The point is how the decision process stays grounded.

## Raw Idea

"There should be something that helps solo builders stop drifting and choose what to do next."

This is directionally interesting, but still vague.

## Three-Line Clarification

- who is this for: solo builders with limited budget, time, and too many AI tools
- what painful problem does it solve: they confuse motion with progress and keep drifting between ideas, tools, and features
- what is the smallest useful version: a practical system that helps them choose the next sane step from idea to live app

Now the project is clearer.

## Spec-Lite

### Core user flow

1. builder arrives with a vague idea
2. builder uses a simple framework to scope it
3. builder defines a first milestone
4. builder ships something small
5. builder reviews and chooses the next step

### Must-haves

- clear doctrine
- concrete workflow
- project brief template
- milestone review template
- deploy checklist

### Not now

- user accounts
- dashboards
- collaboration features
- analytics
- AI integration layer

### Stack choice

Docs-first repo.

No app needed yet.

### Milestone 1

Create a public open-source repo that contains the first usable version of the system.

## What Got Built

Milestone 1 produced:

- README with project positioning
- philosophy doc
- workflow doc
- start-here guide
- project brief template
- milestone review template
- deploy checklist

## Why This Is a Good Milestone

It proves:

- the idea can be expressed clearly
- the system can exist without software complexity
- the repo already has practical value

It does not try to prove:

- long-term traction
- monetization
- app requirements
- full category fit

That is correct.

## What Validation Looks Like Next

Reasonable next signals:

- the creator uses Grounded on a real project
- another solo builder can follow it without confusion
- the templates help narrow a vague idea into a real milestone

That is enough for the next step.

## What the Next Sane Step Might Be

Not:

- build a SaaS
- add authentication
- create a polished dashboard

Better:

- add a canonical guide
- add an audience-fit document
- add one more concrete walkthrough

That is how Grounded grows without drifting.
